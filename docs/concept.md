## ミッション

> **QuuBee は、PC-98 のフリーソフト文化を、罪悪感なく継承・再体験できるブラウザプレイヤー。**
>
> NEC BIOS も MS-DOS も使わず、書庫ファイルをドロップすれば、あの頃のように、すぐ遊べる。

タグライン候補:
- 「書庫があれば、すぐ遊べる。あの頃のまま。」
- 「PC-98 フリーソフト文化を、罪悪感なく。」
- 「機械じゃなく、ソフトを継承する。」

呼び方の原則（2026-06-11 決定）: **「エミュレータ」と自称しない**。一人称は「ブラウザプレイヤー」
（エンジン NP2kai の帰属は隠さず表示する — 技術の出自でなく、プロダクトの自己定義の問題）。
サイトは**思想先行** — クリーンと敬意を、使い方より先に見せる。初回訪問時に設計思想の
**宣言**（日英、`web/index.html` の `#about-text`）をポップアップ表示（2026-06-13 実装）。

## 3 本の柱

### ① 著作権クリーン（構造的な独自性）
ユーザーが触れるスタック全体が著作権クリーン:
- **BIOS**: NEC ROM 不使用。NP2kai 内蔵の最小 BIOS（`nosyscode` + C ハンドラ）+ 合成 BIOS。
- **DOS**: MS-DOS 不使用。INT 21h を C 側で **HLE 模倣**（ミニ DOS ローダ）。
- **フォント**: NEC フォント ROM 不使用。`font.bmp`（修正 BSD）で代替。帰属は [CREDITS.md](../CREDITS.md)。
- **コア**: NP2kai（MIT）を **LLE** で利用。

→ **知財保護と互換性を両立**。結果として **「URL を開くだけ」で合法に成立する、公開ホスティング可能な
PC-98 系プレイヤー**になり得る（実機 BIOS/DOS を要求する他のエミュは公開配布が法的に困難）。

### ② お手軽さ（当時の流儀の再現）
当時のフリーソフト体験は「Vector/Nifty/BBS で書庫を入手 → LHA で解凍 → EXE を叩く → 即遊ぶ」。
QuuBee の「**書庫をドロップ → 即プレイ**」はこれそのもの。ディスクイメージ作成も DOS インストールも
config.sys いじりも不要。「**機械を丸ごと立ち上げる**」のではなく「**ソフトを動かす**」。

### ③ 敬意（罪悪感なく、を支える仕組み）
- **フリーソフト限定**（商用ゲームを動かす公開経路は作らない）。
- **原作者クレジット・連絡先・readme をそのまま継承**（テキストビューアで提示）。
- **Tier-1 内蔵は許諾ベース**（原作者の許可を得たフリーソフトのみ）。
- **第三者アセットのライセンスを正しく帰属**（同梱フォント等、[CREDITS.md](../CREDITS.md)）。

> 削除依頼（オプトアウト）窓口は置かない。QuuBee はゲームを配布・ホストしない（Tier-1 は許諾ベース、
> Tier-2 はユーザー自身のファイル）ため、「削除依頼」はホスティング事業者の概念で QuuBee には当てはまらない。
> リストからの除外等が要れば連絡ベースで個別対応すれば足りる。

## 「快適化」の意味

QuuBee の「快適化」は **「ハードの制約からの解放」を CPU 速度だけでなくコンテンツアクセス全般へ広げる**こと:

- **CPU**: 適応オートクロック（クロック倍率を負荷に応じ 20〜42 で自動調整、既定 ON。
  VSync/タイマ依存ゲームは無調整で快適化）。
- **ディスク入れ替え不要**: 複数ディスクのフリーソフトも全ファイルを 1 ディレクトリに展開して起動
  （当時の入れ替え地獄が消える ＝ 作者が本来させたくなかったはずの体験の解放）。
- **即起動・即解凍**: FDD 待ちなし、起動シーケンス・スキップ。

互換性は手段、快適化は機能、**文化継承が目的（魂）**。

## 入口ポリシー（窓口）

### 正面玄関＝書庫ファイル一本
- **対応: `.lzh`（当時の本流）+ `.zip`（Vector/Internet Archive の現代チャネル）**。RAR は対象外。
- **複数書庫の同時展開**: 複数の `.lzh`/`.zip` を一度に投入 → すべて同じ仮想 HD ディレクトリ
  （Emscripten FS `/run/`）へ展開 ＝ **デジタル版「HD にインストール」**。多ディスクの大作フリー RPG 等に対応。
- **パッチ（2 形態に対応）**:
  - (a) **ファイル上書き型** — 「本体書庫 + パッチ書庫」を順に展開（last-wins、展開順=投入順、並べ替え可）。
  - (b) **パッチャ EXE 型** — 「`PATCH.EXE` を実行して対象のバイナリを書き換える」もの。QuuBee は実 DOS
    プログラムを動かせるので**パッチャ EXE をそのまま実行**でき、`/run/` のファイルを書き換える（INT 21h の
    open/seek/write）。

### ディスクイメージの扱い（赤線＝ユーザーのディスクイメージからのブート）
ディスクイメージは**ファイルコンテナとして**のみ扱う。引くべき赤線は **「ユーザーが持ち込んだ
ディスクイメージから *ブート* させること」**（＝商用ゲームの丸ごとダンプ／機械丸ごと起動の経路）。
- **取り出しは実装済み**（`web/player/diskimage.js`、2026-06-03）: D88/FDI/DCP/raw 等を**ブートせず**、
  中の FAT12/16 ファイルだけ `/run/` へ展開。自己起動・非 FAT のイメージは明示メッセージで弾く。
  プロテクト保全用フォーマット（NFD/BKDSK/VFDD）は**恒久対応外**（保全はミッションと逆向き）。
- **サイトにブート機構そのものが無い**: 旧ディスクブート経路（任意イメージの挿入+リセット）は
  2026-06-12 の棚卸しでコードごと撤去した。残る内部ブートはミニ DOS ローダ自身の `loader.d88`
  （毎 Run、固定内容を pristine 再生成）のみ ＝「窓口を絞る」から進めて「機構を持たない」へ。
- **真の自己起動もの**（例: Super Depth `.fdi`）はブートが必須なので、公開では**書庫（`.lzh`）版を使う**。

## プロダクト層（フロントエンドの "皮"）

重要原則: **著作権クリーンなコア（Wasm）は完成・安定。プロダクト機能はすべて JS 側**で、Wasm は触らない。
2026-06-12 のユーザー判断「フリーソフトの実行環境としてはベース完成」以降、能動投資はこの層に移行
（互換性は不具合駆動の受け身）。

実装済み:
- **テキスト/画像ビューア**: readme 等の SJIS 表示（NEC 罫線→Unicode 写像で罫線/AA を保全）、
  別窓拡大（実機風の行間ゼロ + VZ Editor 慣習の %X タグをクリックジャンプに翻訳）、PC-98 標準
  `.MAG` (MAKI02) / `.PI` (Pi) 画像の自前デコーダ。文書 ＝ 文化的アーティファクト（作者の声・謝辞・連絡先）の
  継承であり、③敬意を体現。**readme/VZ 慣習/MAG という周辺文化の再体験は QuuBee の差別化軸**
  （2026-06-12 確認）。
- **ファイラ UI**: ヘッダ状態機械（Open/＋Add/×）、パンくず、種別アイコン、選択 2 軸
  （行背景=タップ中 / 緑チップ=Run 対象）。ラベル/ステータスは英語、散文は日本語。
- **ファイル単体の Save / ＋Add 読み戻し**: 出入りとも **ASCII 8.3 名限定の対称ルール**
  （「Save できたものは必ず戻せる」保証）。セーブの往復永続化・自作データの持ち込み/回収。
- **起動 .bat のレシピ解釈** + エントリ自動検出（中央 DB 不要。開発テスト用だった `db/games.json` は
  削除済み、2026-06-12）。
- **宣言（About）**: 初回訪問時に設計思想を日英ポップアップ表示（2026-06-13）。

方針確定（2026-06-11 ユーザー決定）:
- **`.qb` 可搬ファイル（原書庫内蔵の保存形式）は断念** — 自己記述的で保存・流通に都合がよい反面、
  **原書庫の再配布を誘う**ためコンセプト（③敬意）と両立しない。
- **永続化はブラウザストレージのみ**（IndexedDB 等、ユーザーの手元に閉じる）。
  「変更分だけ zip 書き出し」＝状態のみエクスポート案は Save/＋Add の段階 2 候補（採否未決）。
- **中央フィンガープリント DB は持たない**: readme の継承表示 + エントリ自動検出 + .bat レシピ解釈で
  同定・帰属・起動知識をカバーできている。

未着手/検討中:
- **本棚（個人ローカルライブラリ）+ サムネイル**: ブラウザストレージ前提で再設計
  （`.qb` 前提だった旧案は破棄。サムネは canvas スナップショット `toBlob` の見込み）。
- **Tier-1 内蔵**: 作者公認フリーソフトのプリシード（許諾ベース）。

### 体験ループ（現行）
> 書庫をドロップ → readme を読みながらエントリ自動検出 / .bat レシピで即プレイ →
> セーブ等は [Save] で書き出し、次回 [＋Add] で読み戻し（本棚・常設永続化はコンセプト練り直し待ち）

## やらないこと

- ❌ CRT シェーダ / 走査線エフェクト（当時の質感の再現はしない）
- ❌ 起動シーケンスを演出として再現すること（凝った BIOS POST 画面など）。
  ※ 高速メモリチェック由来の一瞬の表示は速くて無害なので、あえて抑止はしない。
- ❌ エミュレータ設定 UI（CPU/RAM/サウンドボード選択）
- ❌ 機種切替（**PC-9821/NP21 に固定**）
- ❌ 詳細なキーマッピング設定 UI
- ❌ スナップショット式セーブステート（ゲーム本来のセーブ機構を使う。ただしそのセーブと本棚は永続化する）
- ❌ FM 音楽のエクスポート等の技術的機能
- ❌ ネイティブアプリ化（ブラウザのまま、PWA は可）
- ❌ 持ち込みファイルの設定手動チューニング UI
- ❌ 商用ゲームを動かす公開経路（ディスクイメージからブートする機構をサイトに持たない）
- ❌ `.qb` 可搬ファイル（原書庫内蔵の保存形式）— 再配布を誘うため断念（2026-06-11）
- ❌ 削除依頼（オプトアウト）窓口（配布・ホストしないため不要）

## 対象スコープ

- **フロッピーベース・2D・〜1998 年の PC-98 同人/フリーソフト**。カバー率見込み **80〜90%**。
- 対象外: DOS Extender（DOS/4GW, PMODE/W; DPMI host が必要）、CD-ROM（MSCDEX/ISO9660）、
  N88-BASIC ROM 依存（著作権）、商用ソフト全般、分割書庫（`.l01`/`.l02`）。
- **3D ポリゴンは対象外ではない**: PC-98 に 3D ハードは無く、当時の「3D」は全て**ソフトウェアレンダリング**。
  QuuBee は i386c + FPU(DOSBox2) を持つので CPU/FPU で普通に動く。「対象外」は欠けているサブシステム
  （DPMI/MSCDEX 等）の話で、3D は該当しない。
- 実用ソフト（一太郎/Lotus/桐 等）は技術的に動作可能だが、フリー/許諾が前提。
